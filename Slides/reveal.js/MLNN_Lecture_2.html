Decision Trees

  1. A decision tree represents a function that takes as input a vector of attribute values and returns a decision (a single output value).
  2. It reaches its decision by performing a sequence of tests.
  3. Each internal node corresponds to a test of the value of one of the input attributes \(A_i\) and the branches from the node are labeled with the possible values of the attribute.
  4. Each leaf node in the tree specifies a value to be returned by the function.
  5. For many problems, the decision tree format yields a nice, concise result .

S. Russell and P. Norvig. Artificial Intelligence. A Modern Approach. Third Edition. Pearson Press. 2010.

Learning decision trees from examples

  1. The algorithm to learn the decision trees (\(DT\_Learning\)) adopts a greedy divide-and-conquer strategy: always test the most important attribute first.
  2. The test divides the problem up into smaller problems that can be solved recursively.
  3. The most important attribute is the one that makes the most important difference to the classification of an example.
  4. The goal is to get to the correct classification with a small number of tests.
    5.
  5. To identify the most important attributes, different importance measures are defined.
    6.

S. Russell and P. Norvig. Artificial Intelligence. A Modern Approach. Third Edition. Pearson Press. 2010.

Supervised learning

Learning decision trees: Split metrics

  1. At each step, the DT learning algorithm finds the pair of attribute and cutting point which makes the maximal impurity decrease.
  2. Let \(p_i\) denote the probability of a class computed from the observations at a given node.
  3. Gini impurity: \[ 1- \sum_{i=1}^{c} p_i^2 \]
  4. Entropy: \[ - \sum_{i=1}^{c} p_i^2 log_2(p_i) \]
  5. Classification error: \[ 1 - \max_{i \in \{1 \dots c\}} p_i \]

Learning decision trees: Variants of the algorithm according to split criteria

  1. Iterative Dichotomiser 3 (ID3) algorithm is based on Shannon entropy.
  2. C4.5 algorithm is based on Gain Ratio which is considered as a normalized Shannon entropy.
  3. Classification And Regression Tree (CART) algorithm is based on Gini index.

Supervised learning

Bootstrap Aggregating (Bagging)

  1. Sets of observations (bootstrap samples) are chosen from the data with replacement (some observations can be in more than one bootstrap sample).
  2. From each bootstrap sample, a classifier is learned.
  3. The predictions computed from all samples are combined.
    4.
  4. It is a variant of ensemble classifiers, is a method to reduce the variance of the prediction.
    5.
  5. When used with decision trees, usually the same greedy algorithm is used to learn each tree, producing similar split points.
    6.

L. Breimann. Bagging predictors. Machine Learning. Vol. 24 No. 2. Pp. 123--140. 1996.

Supervised learning

Random forests

  1. A random forest is a set of decision trees each one learned on a bootstrap sample.
  2. In each sample, at the time of learning a tree, a subset of all variables is selected to select the best split.
    3.
  3. A large number of tree is generally used.
  4. Notably improves the accuracy of decision trees classifiers.
  5. However, the interpretability of the random forest is very limited.
    6.

L. Breimann. Random Forests. Machine Learning. Vol. 45 No. 1. Pp. 5--32. 2001.

Supervised learning

Random forests: Variable importance

  1. Variable importance: A relevant concept in machine learning. It is a measure of how important each variable is for the classification process.
  2. Two variable importance measures are usually computed from random forests:
    • Based on the Gini, or other information-based measures.
    • Based on permutations of the variables in the trees and the quantification of their effect on the classification error.
  3. Local variable importance can be also computed since son variables are relevant to classify subsets of observations.

Support vector machines

  1. Similar to logistic regression, it is driven by a linear function \(w^T{\bf{x}}+b\).
  2. Based on the concept of margin, which is defined to be the smallest distance between the decision boundary and any of the samples.
  3. The decision boundary is chosen to be the one for which the margin is maximized.
  4. Support vectors are the elements of the training set that would change the position of the dividing hyperplane if removed.
  5. A kernel function is used to evaluate the decision function.

C. M. Bishop Pattern Recognition and Machine Learning.. Springer. 2009.

Support vector machines

Kernel based classification

  1. The kernel trick: Many ML algorithms can be written exclusively in terms of dot products between examples. For example, for the linear function used by SVM: \[ w^{\top} {\bf{x}} + b = b+ \sum_{i=1}^{m} \alpha_i {\bf{x}}^{\top} {\bf{x}}^i \]
  2. This can be further generalized replacing \( {\bf{x}}\) bythe output of a given feature function \( \phi( {\bf{x}})\).
  3. The dot product is also replaced with a function \( k( {\bf{x}}, {\bf{x}}^i)= \phi( {\bf{x}})\phi( {\bf{x}}^i)\) called a kernel.

I. Goodfellow and Y. Bengio and A. Courville. Deep Learning.. MIT Press. 2016.

C. M. Bishop. Pattern Recognition and Machine Learning.. Springer. 2009.

Support vector machines

Kernel based classification

  1. In SVM predictions are made using the function: \[ f({\bf{x}}) = b + \sum_{i=1}^{m} \alpha_i k({\bf{x}},{\bf{x}}^i) \]
  2. This function is nonlinear with respect to \({\bf{x}}\) but the relationship between \(\alpha\) and \(f({\bf{x}})\) is linear.
  3. The model is equivalent to preprocessing the data by applying \( \phi( {\bf{x}})\) to all inputs, then learning a linear model in the new transformed space.

I. Goodfellow and Y. Bengio and A. Courville. Deep Learning.. MIT Press. 2016.

C. M. Bishop. Pattern Recognition and Machine Learning.. Springer. 2009.

Support vector machines

Extensions

  1. Multiclass SVMs: Construct \(K\) separate SVMs in whith the k\(^{th}\) model \(y_{k}({\bf{x}})\) is trained using the data from class \(C_{k}\) as the positive examples and the data from the remaining \(K\) as negative examples.
  2. SVMs for regression: The quadratic error function is replaced by an \(\epsilon\)-insensitive error function between the prediction and the target which gives zero error if the absolute difference between the prediction \(y({\bf{x}})\) and the target \(t\) is less than \(\epsilon\), where \(\epsilon > 0\).

C. M. Bishop. Pattern Recognition and Machine Learning.. Springer. 2009.

V. N. Vapnik. Statistical Learning Theory.. Wiley. 1998.

Support vector machines

Limitations

  1. The ouputs of an SVM represent decisions rather than posterior probabilities.
  2. The cost of evaluating the decision function for kernel machines is linear in the number of training examples, although SVM set many of the $\alpha_i$ coefficients to zero.
  3. There is a complexity parameter \(C\) (or parameter\( \nu \) ) that has to be estimated as part of the classifier.
  4. The extension to \(K\) classes is problematic since SVM was originally conceived for binary classification problems.
  5. Predictions are expressed as linear combinations of kernel functions that are centred on trainin data points.

C. M. Bishop. Pattern Recognition and Machine Learning.. Springer. 2009.

Evaluating classifiers

k-fold crossvalidation

  • Used to estimate the classifier performance (e.g., accuracy) when a validation set is not available.
  • The dataset is randomly partitioned into k equal sized folds.
  • The cross-validation process is repeated k times. Every time instances in a different fold are predicted.
  • Leave-one-out cross-validation (LOOCV): Each fold corresponds to an instance. In every step, one instance is leave out for training and its class predicted.

Classifiers

The three components of learning algorithms

  1. Representation: Choosing a representation for a classifier influence to a large extent the set of classifiers that it can learn. This set is called the hypothesis space of the learner.
  2. Evaluation: To distinguish between good and poor classifiers at least one criterion is required that is usually evaluating using an objective function or scoring function.
  3. Optimization: The search for the best classifier in the hypothesis space is usually posed as an optimization problem. The choice of the optimization technique is key to the efficiency of the learner.

P. Domingos. A few useful things to know about machine learning. Communications of the ACM, 55(10), 78-87. 2012.

Learning the models

Support vector machines

  1. We will to optimize the parameters \(w\) and \(b\) so to maximize the distance of the closest point \(x^n\) to the decision surface: \[ argmin_{w,b} \left\{ \frac{1}{||w||} min_n [t^n (w^{\top} \phi({\bf{x}}^n) +b) ]\right\} \]
  2. This problem can be transformed to \( argmin_{w,b} \frac{1}{2}||w||^2 \) subject to \({\bf{y}}^i(w \phi({\bf{x}}^i)+b) \geq 1 \)
  3. An this constrained problem can be posed as a Lagrangian optimization problem: \[ argmin_{w,b} \frac{1}{2}||w||^2 - \sum_i \alpha_i ({\bf{y}}^i( \phi(w{\bf{x}}^i)+b) - 1) \]
  4. This is a quadratic programming optimization problem.

C. M. Bishop. Pattern Recognition and Machine Learning.. Springer. 2009.
